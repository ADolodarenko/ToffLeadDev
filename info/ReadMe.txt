Версия 1.0.1.0:
1. Доработана нормализация телефонного номера.
В нем программа избавляется от всех нечисловых символов, оставляет 10 последних символов и добавляет лидирующиe символы, указанные в настройках (см. далее) в том случае, если обработанный номер имеет длину 10 символов;
2. Реализованы загрузка, редактирование и сохранение следующих настроек программы:
Main Application Title - Заголовок главного окна программы. Первоначальное значение - Toff Lead Developer;
Default API URL - URL-адрес банковского сервиса по умолчанию. Первоначальное значение - https://origination.tinkoff.ru/api/v1/public/partner/);
Phone prefix - префикс телефонного номера - символы, которые программе следует добавлять к 10-ти последним цифрам телефонного номера в ходе его нормализации перед отправлением заявки (например - "+7", "8"). Первоначальное значение - 8;
API: Lead Product - служебное поле заявки согласно API (см. API-документацию. Меняется по согласованию с банком). Первоначальное значение - РКО;
API: Lead Source - служебное поле заявки согласно API (см. API-документацию. Меняется по согласованию с банком). Первоначальное значение - Локальные агенты;
API: Lead Subsource - служебное поле заявки согласно API (см. API-документацию. Меняется по согласованию с банком). Первоначальное значение - API;
API: Lead Temperature - служебное поле заявки согласно API (см. API-документацию. Меняется по согласованию с банком). Первоначальное значение - COLD;
API: Lead Is Hot - служебное поле заявки согласно API (см. API-документацию. Меняется по согласованию с банком). Первоначальное значение - False.
3. В целях безопасности поля agent-id, api-key и api-secret в интерфейсе программы больше не заполняются автоматически.

Программа "Toff Lead Developer" предназначена для загрузки данных по формату из файла XLS или XLSX и отправки заявок на обслуживание на сайт банка Tinkoff по этим данным.
Программа требует наличия платформы .NET 4.5 для корректной работы.
Программа имеет визуальный интерфейс, состоящий из одного окна, где указывается путь к исходному файлу и параметры подключения к сайту. На старте приложения в его окне прописываются параметры по умолчанию, которые в дальнейшем можно редактировать.
Также в главном окне находится кнопка для запуска процесса загрузки и отправки и поле лога работы программы, где находятся внутренние ее сообщения и сообщения сайта банка.

Описание формата исходных данных.
Программа читает данные, расположенные на 1-м листе исходного XLS(X)-файла, начиная с 1-й строки. Соответственно, заголовка для данных быть не должно.
Впрочем, если он будет, программа выдаст свое внутреннее сообщение о некорректных данных соответствующей строки и продолжит работу.
Любой исходный файл должен иметь единый формат. Поля, перечисленные в требуемом порядке:
ИНН - согласно документации по API банка Tinkoff 10, 12, 13, 15 цифр - обязательное;
Фамилия - обязательное;
Имя - не обязательное;
Отчество - не обязательное;
Телефон - принимаются в расчет номера в виде +7XXXXXXXXXX / 8XXXXXXXXXX / XXXXXXXXXX, где X - цифра - обязательное;
Наименование организации - если отсутствует, будет указано, как "ИП <Фамилия> <Имя> <Отчество>" - не обязательное;
E-mail - не обязательное.

Директоря Example содержит переданные для примера исходные файлы и соответствующие им файлы по формату:
ИП для апи Р.xlsx -> Example01.xlsx
ИП для апи А.xlsx -> Example02.xlsx
ооо для API Р.xls -> Example03.xls
